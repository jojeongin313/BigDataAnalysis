## 사전 준비

1. **Kaggle 원본 데이터 배치**
   - 다운로드: [Kaggle CSIC 2010 Web Application Attacks](https://www.kaggle.com/datasets/ispangler/csic-2010-web-application-attacks)
   - 저장 위치: `data/archive/csic_database.csv`

2. **수업용 CSV 생성**: `python download_csic2010.py`
   - Kaggle CSV 컬럼 정규화 + 1만 건 샘플링 (라벨 비율 유지)
   - 결과: `csic2010_requests.csv` (1만 건 × 18 컬럼)
   - 전체 6만 건을 쓰려면: `python download_csic2010.py --full`

3. **JupyterLab 실행**: `jupyter lab`
   - 브라우저에서 `BigDataAnalysis/7_week/` 폴더로 이동하여 `.ipynb` 파일 더블클릭

## 실습 순서

### 실습 3: data_load_explore.ipynb
- `pd.read_csv()`로 정제된 HTTP 요청 데이터 로드
- 데이터 크기, 컬럼 목록 확인
- **핵심**: `label` 분포 확인 → Normal(~59%) vs Anomalous(~41%)
- HTTP 메서드별 분포 (GET/POST/PUT)
- 실제 정상/공격 HTTP 요청 텍스트를 눈으로 직접 확인
- URL 길이 기본 통계 (정상 vs 공격 차이)

### 실습 4-A: eda_visualization.py / eda_visualization.ipynb ★
- **발표용**: Streamlit 웹 앱 (`streamlit run eda_visualization.py`)
- **분석용**: JupyterLab 노트북 (`eda_visualization.ipynb`) — Plotly 차트를 셀 단위로 학습
- 4개 영역에서 데이터를 다양한 각도로 탐색:
  1. **분포 분석**: 정상/공격 파이 차트 + 메서드별 분포
  2. **길이 분석**: URL 길이, POST Body 길이 히스토그램/Box Plot
  3. **공격 키워드**: SQL/XSS/Path Traversal 등 키워드 빈도 분석
  4. **HTTP 요청 뷰어**: 정상/공격 요청 텍스트 직접 비교 + 키워드 검색

### 실습 4-B: eda_keyword_analysis.ipynb
- 공격 카테고리별 키워드 빈도 (SQL Injection / XSS / Path Traversal / Command Injection / CRLF)
- URL 인코딩 패턴 분석 (`%27`, `%3C` 등)
- 가장 많이 접근된 경로 Top 10

## 관찰 포인트

학생들에게 아래 질문을 던지며 탐색하도록 유도:

1. **공격 요청의 URL 길이가 정상보다 긴 이유는?** → 공격 코드가 삽입되므로
2. **SQL Injection 키워드는 정상 요청에도 나타나는가?** → 키워드 탭에서 확인
3. **POST vs GET 중 어디에 공격이 더 많은가?** → 메서드별 분포 확인
4. **HTTP 요청 뷰어에서 공격 패턴을 직접 찾아보세요** → 1교시 이론과 연결
