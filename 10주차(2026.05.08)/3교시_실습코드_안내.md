## 실습 순서 (반드시 순서대로!)

### 실습 5: preprocessing.ipynb ★★
- URL 디코딩: `%27` → `'`, `%3C` → `<` (Latin-1 인코딩)
- **특성 엔지니어링** — HTTP 텍스트에서 21개 숫자형 특성 직접 추출:
  - 길이 특성: url_length, body_length, total_length
  - URL 구조: num_params, path_depth, has_query, query_length
  - 공격 키워드: has_sql, has_xss, has_traversal, has_cmd_injection, has_crlf
  - 통계: url_entropy, digit_ratio, upper_ratio, num_special_chars
- **핵심 개념**: 텍스트 → 숫자 변환 = 머신러닝이 학습 가능한 형태로 바꾸기
- 결과 저장: `csic2010_cleaned.csv`

### 실습 6: feature_engineering.ipynb
- 이진 분류 라벨: Normal=0, Anomalous=1
- 특성 선택: 추출된 21개 숫자형 특성
- **StandardScaler**: 학습 데이터로 fit, 테스트는 transform만 (데이터 누수 방지!)
- 학습/테스트 분할 (80:20, stratify로 비율 유지)
- 결과를 `processed_data.pkl`로 저장 → 10주차에서 바로 사용
- LLM 테스트용 샘플 500건 포함 (원본 HTTP 텍스트 포함!)

### 실습 7: approach_preview.ipynb ★
- 3가지 분류 접근법을 소규모 데이터로 빠르게 체험
- **통계(IQR)**: 각 특성별 이상치 개수 합산 → 임계값 이상이면 공격
- **머신러닝(RandomForest)**: 추출된 특성으로 학습 → 특성 중요도 확인
- **LLM(Ollama)**: HTTP 요청 텍스트를 그대로 보여주고 판단 ★
  - CSIC 2010의 강점: LLM이 텍스트를 직접 이해!
- 3가지 방법의 정확도/속도 비교표 확인

## 10주차 예고

`processed_data.pkl`을 로드하여:
1. IQR 이상치 탐지 (url_length, num_special_chars 등 기반)
2. RandomForest + XGBoost 모델 학습 + 특성 중요도 분석
3. Ollama LLM으로 HTTP 요청 직접 분류 (프롬프트 엔지니어링)
4. 3가지 방법의 성능 비교 (정확도, F1, 속도, 혼동행렬)
5. Streamlit 웹 공격 탐지 대시보드 구현
