## 사전 준비

- `csic2010_requests.csv` 파일이 같은 폴더에 있어야 합니다
- 데이터 준비: `python download_csic2010.py`
  - CSIC 2010 데이터셋의 3개 텍스트 파일을 파싱하여 CSV 생성
  - 필요 파일: `normalTrafficTraining.txt`, `normalTrafficTest.txt`, `anomalousTrafficTest.txt`

## 실습 순서

### 실습 3: data_load_explore.py
- `pd.read_csv()`로 파싱된 HTTP 요청 데이터 로드
- 데이터 크기, 컬럼 목록 확인
- **핵심**: `label` 분포 확인 → Normal(~74%) vs Anomalous(~26%)
- HTTP 메서드별 분포 (GET/POST)
- 실제 정상/공격 HTTP 요청 텍스트를 눈으로 직접 확인
- URL 길이 기본 통계 (정상 vs 공격 차이)

### 실습 4: eda_visualization.py ★
- **Streamlit 웹 앱**으로 실행 (`streamlit run`)
- 4개 탭에서 데이터를 다양한 각도로 탐색:
  1. **분포 분석**: 정상/공격 파이 차트 + 메서드별 분포
  2. **길이 분석**: URL 길이, POST Body 길이 히스토그램/Box Plot
  3. **공격 키워드**: SQL/XSS/Path Traversal 등 키워드 빈도 분석
  4. **HTTP 요청 뷰어**: 정상/공격 요청 텍스트 직접 비교 + 키워드 검색

## 관찰 포인트

학생들에게 아래 질문을 던지며 탐색하도록 유도:

1. **공격 요청의 URL 길이가 정상보다 긴 이유는?** → 공격 코드가 삽입되므로
2. **SQL Injection 키워드는 정상 요청에도 나타나는가?** → 키워드 탭에서 확인
3. **POST vs GET 중 어디에 공격이 더 많은가?** → 메서드별 분포 확인
4. **HTTP 요청 뷰어에서 공격 패턴을 직접 찾아보세요** → 1교시 이론과 연결
