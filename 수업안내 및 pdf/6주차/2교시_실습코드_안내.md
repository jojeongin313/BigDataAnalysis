# 6주차 2교시 — 프롬프트 엔지니어링 & 텍스트 분석

## 실습 코드 목록

| 실습 | 파일명 | 내용 | 실행 방법 |
|------|--------|------|----------|
| Step 1 | `prompt_compare.py` | 나쁜 프롬프트 vs 좋은 프롬프트 비교 | `python prompt_compare.py` |
| Step 2 | `prompt_role.py` | 역할 부여 패턴 (CEO / 교수 / 노동자) | `python prompt_role.py` |
| Step 2 | `prompt_fewshot.py` | Few-shot 리뷰 카테고리 분류 | `python prompt_fewshot.py` |
| 실습 7 | `llm_sentiment.py` | LLM으로 영화 리뷰 감성 분석 | `python llm_sentiment.py` |
| 실습 8 | `llm_analysis.py` | 키워드 추출 & 3줄 요약 | `python llm_analysis.py` |
| 실습 9 | `llm_json.py` | JSON 구조화 출력 + 파싱 | `python llm_json.py` |
| 실습 10 | `ollama_stream.py` | 스트리밍 출력 (한 글자씩 실시간) | `python ollama_stream.py` |
| 실습 11 | `hf_vs_ollama.py` | HuggingFace Pipeline vs Ollama 비교 | `python hf_vs_ollama.py` |

## 사전 준비

1. **Ollama 서버 실행 확인**
2. **Python 패키지 설치** — `pip install ollama transformers torch`
3. 실습 11(`hf_vs_ollama.py`)은 HuggingFace 모델을 다운로드하므로 **인터넷 연결 필요** (최초 1회)

## 실습 순서

### prompt_compare.py
- 같은 주제("파이썬")에 대해 모호한 프롬프트 vs 구체적인 프롬프트로 결과 비교
- 좋은 프롬프트의 5가지 원칙 체험: 역할 부여, 구체적 지시, 출력 형식, 예시, 제약 조건

### prompt_role.py
- 동일한 질문("인공지능이 일자리를 빼앗을까요?")에 3가지 역할 부여
- 역할에 따라 관점이 완전히 달라지는 것을 관찰

### prompt_fewshot.py
- Few-shot 학습: 프롬프트에 입출력 예시 4개를 포함
- 모델이 패턴을 파악하여 새로운 리뷰를 올바르게 분류하는지 확인

### llm_sentiment.py
- 4개 영화 리뷰를 LLM으로 감성 분석 (긍정/부정/중립)
- 시스템 프롬프트로 "감성 분석 전문가" 역할 부여

### llm_analysis.py
- 의료 AI 관련 기사에서 키워드 5개 추출 + 3줄 요약
- 출력 형식을 프롬프트로 제어하는 방법 체험

### llm_json.py
- LLM에게 JSON 형식으로만 응답하도록 지시
- `json.loads()`로 파싱하여 프로그램에서 활용하는 방법
- LLM이 ```json 코드블록으로 감싸는 경우 처리하는 파싱 로직 포함

### ollama_stream.py
- `stream=True` 옵션으로 실시간 스트리밍 출력
- `flush=True`로 한 글자씩 나타나는 효과 체험

### hf_vs_ollama.py
- 동일한 감성 분석 작업을 HuggingFace Pipeline과 Ollama LLM으로 수행
- 속도, 출력 형식, 정확도 차이를 직접 비교

## 주의사항

- `llm_sentiment.py`, `llm_analysis.py`는 여러 번 LLM을 호출하므로 시간이 걸릴 수 있음
- `llm_json.py`에서 LLM이 항상 완벽한 JSON을 출력하지 않을 수 있음 → `try/except`로 처리
- `hf_vs_ollama.py`는 HuggingFace 모델 최초 다운로드 시 시간 소요
