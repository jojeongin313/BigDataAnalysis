# 6주차 3교시 — Streamlit + Ollama/HuggingFace 챗봇 구현

## 실습 코드 목록

| 실습 | 파일명 | 내용 | 실행 방법 |
|------|--------|------|----------|
| 실습 12 | `chat_ui_basic.py` | Streamlit 채팅 컴포넌트 기본 구조 | `streamlit run chat_ui_basic.py` |
| 실습 13 | `chatbot_basic.py` | session_state로 대화 기록이 유지되는 Ollama 챗봇 | `streamlit run chatbot_basic.py` |
| 실습 14 | `chatbot_stream.py` | 스트리밍(실시간 타이핑) Ollama 챗봇 | `streamlit run chatbot_stream.py` |
| 실습 15 | `chatbot_full.py` | 시스템 프롬프트 + 사이드바 + Temperature 완성형 챗봇 | `streamlit run chatbot_full.py` |
| 실습 16 | `hf_korean_gen.py` | HuggingFace 한국어 텍스트 생성 (터미널) | `python hf_korean_gen.py` |
| 실습 17 | `chatbot_hf_korean.py` | HuggingFace 한국어 모델 Streamlit 챗봇 | `streamlit run chatbot_hf_korean.py` |
| 실습 18 | `chatbot_combined.py` | Ollama + HuggingFace 통합 비교 챗봇 | `streamlit run chatbot_combined.py` |

## 사전 준비

1. **Ollama 서버 실행 확인** (실습 13~15, 18에서 필요)
2. **Python 패키지 설치** — `pip install ollama streamlit transformers torch`
3. 실습 16~18은 HuggingFace 모델(`skt/ko-gpt-trinity-1.2B-v0.5`, 약 2.5GB)을 다운로드하므로 **인터넷 연결 필요** (최초 1회)

## 실습 순서

### 실습 12: chat_ui_basic.py
- `st.chat_message()`와 `st.chat_input()` 채팅 전용 컴포넌트 체험
- 입력하면 이전 메시지가 사라지는 문제 관찰 → 다음 실습에서 session_state로 해결

### 실습 13: chatbot_basic.py
- `st.session_state.messages`에 대화 기록을 누적 저장
- Ollama에 전체 messages 리스트를 전달하여 이전 대화를 기억하는 챗봇 구현
- Streamlit의 Rerun 동작과 session_state의 관계 이해

### 실습 14: chatbot_stream.py
- `stream=True` + `st.write_stream()`으로 실시간 타이핑 효과 구현
- 제너레이터(Generator) 함수를 사용하여 스트리밍 데이터를 화면에 전달

### 실습 15: chatbot_full.py
- 사이드바에서 AI 역할 선택 (일반 대화 / 파이썬 튜터 / 데이터 분석가 / 영어 선생님)
- 직접 시스템 프롬프트 입력 가능
- Temperature 슬라이더로 창의성 조절
- 대화 초기화 버튼
- **주의**: 역할을 변경한 후에는 반드시 [대화 초기화] 버튼을 눌러야 함

### 실습 16: hf_korean_gen.py
- HuggingFace `text-generation` 파이프라인으로 한국어 텍스트 생성
- `skt/ko-gpt-trinity-1.2B-v0.5` 모델 사용
- Base Model(이어쓰기)의 동작 방식 체험
- **이 파일은 `python`으로 실행** (Streamlit 아님)

### 실습 17: chatbot_hf_korean.py
- HuggingFace 한국어 모델을 Streamlit 챗봇으로 구현
- `@st.cache_resource`로 모델을 캐싱하여 매번 로드하지 않도록 최적화
- 입력 부분을 제거하고 생성된 부분만 표시하는 로직 포함
- 실습 가이드: "오늘 날씨가 좋아서", "인공지능의 미래는" 등 입력 후 이어쓰기 확인

### 실습 18: chatbot_combined.py
- Ollama와 HuggingFace 모델을 하나의 앱에서 전환하며 비교
- 사이드바 라디오 버튼으로 모델 선택
- 같은 입력에 대해 Instruction-tuned(대화형) vs Base Model(이어쓰기) 차이 체험

## 주의사항

- Streamlit 앱은 `streamlit run 파일명.py`로 실행 (`python`이 아님)
- 실습 16(`hf_korean_gen.py`)만 `python`으로 실행
- HuggingFace 모델 최초 로드 시 다운로드 시간 소요 (약 2.5GB)
- Ollama와 HuggingFace 모델을 동시에 실행하면 VRAM이 부족할 수 있음
  - 하나를 테스트할 때 다른 프로세스 종료 권장
  - HuggingFace 모델은 `device="cpu"`로 설정되어 있으므로 CPU(RAM)에서 실행됨
